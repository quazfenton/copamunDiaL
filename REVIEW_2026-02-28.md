## Executive Summary

**PlayMate** (copamundiaL) is a comprehensive sports team management application built with Next.js 15, featuring real-time communication via Socket.IO, team management, match scheduling, league management, and a unique **formation builder** with drag-and-drop capabilities.

### 🔴 Critical Finding: Formation Builder is Undervalued
The **formation builder** is the project's most unique and differentiating feature, yet it's buried inside a generic "sports management" app. This is like having a world-class chess engine inside a calendar app.

### 💡 Strategic Pivot Recommendation
**"LineupLab"** - Focus exclusively on the formation/strategy niche. This is a smaller but more winnable market with higher willingness to pay from coaches.

### Market Opportunity
- Sports management software market: **$16.76B by 2032** [^1]
- Youth sports tech investment surge in 2025 [^2]
- Formation tools are rare - most competitors focus on scheduling/communication

import Stripe from 'stripe';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

export async function POST(req: Request) {
  const { leagueId, playerId } = await req.json();
  
  const session = await stripe.checkout.sessions.create({
    payment_method_types: ['card'],
    line_items: [{
      price_data: {
        currency: 'usd',
        product_data: { name: `League Registration: ${league.name}` },
        unit_amount: league.fee * 100,
      },
      quantity: 1,
    }],
    mode: 'payment',
    success_url: `${origin}/league/${leagueId}?success=true`,
    cancel_url: `${origin}/league/${leagueId}?canceled=true`,
  });
  
  return Response.json({ url: session.url });
}

**Problem:** REST endpoints have duplicated logic, no type safety.

**Solution:**
```typescript
// server/routers/team.ts
import { router, procedure } from '../trpc';

export const teamRouter = router({
  list: procedure.query(async ({ ctx }) => {
    return ctx.prisma.team.findMany({
      where: { members: { some: { userId: ctx.session.userId } } },
    });
  }),
  
  createFormation: procedure
    .input(z.object({
      teamId: z.string(),
      formation: z.enum(['4-3-3', '4-4-2', '3-5-2', '4-2-3-1']),
      players: z.array(z.object({ playerId: z.string(), position: z.string() })),
    }))
    .mutation(async ({ input, ctx }) => {
      return ctx.prisma.formation.create({ data: input });
    }),
});
```
